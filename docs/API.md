# API Reference

Interactive OpenAPI docs: **http://localhost:8080/docs** (auto-generated by FastAPI).

| Method | Path | Purpose |
|---|---|---|
| POST | `/aa/upload` | Ingest AA JSON → run full pipeline → scored profile |
| POST | `/predict/income` | Income estimation for a scored customer |
| POST | `/predict/repayment` | FOIR, eligible EMI, per-product capacity |
| POST | `/predict/intent` | 30/60/90-day application probability + reason codes |
| POST | `/predict/risk` | PD, grade A–E, fraud indicators, liquidity risk |
| POST | `/predict/lead` | Fused 0–100 lead score with tier |
| POST | `/predict/explain/{model}` | SHAP drivers (`income`/`intent`/`risk`) |
| POST | `/recommend` | Pre-qualified offers with pricing and reasons |
| POST | `/chat` | Gemini advisor grounded in the customer profile |
| POST | `/whatif` | Scenario simulation: EMI, projected FOIR, verdict |
| GET | `/dashboard` | Executive KPIs, funnel, chart series, lead queue |
| GET | `/customer/{id}` | Full scored profile document |
| GET | `/` | Health + model registry status |

All POST bodies are JSON; customer-scoped calls take `{"customer_id": "CUST00001"}`.
Example:

```bash
curl -X POST localhost:8080/predict/lead \
  -H 'Content-Type: application/json' -d '{"customer_id":"CUST00024"}'
```

`POST /aa/upload` body shape — see `data/samples/sample_aa_payload.json`.
